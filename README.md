# Модуль популярностного отбора Top‑N

## Назначение

Модуль `CourseRecommender` (класс `top_n.py`) формирует список **наиболее популярных курсов** на основе глобальной статистики просмотров, лайков и новизны. Используется как **страховочный слой** для холодного старта (новые пользователи, отсутствие истории) и для дополнения рекомендаций от View Ranker.

## Цель эксперимента по подбору весов

Найти оптимальные коэффициенты `(w_views, w_likes, w_novelty)`, а также параметр `half_life` (период полураспада новизны), чтобы:
- устойчиво нивелировать кликбейт (просмотры – шумный сигнал)
- поднимать новые качественные курсы в топ‑7
- сохранять старые хиты в лидерах
- отодвигать непопулярные курсы в самый низ

## Алгоритм расчёта веса курса

Для каждого курса вычисляется **общий вес**:
weight = views * w_views + likes * w_likes + novelty_score * w_novelty


где  
`novelty_score = scale * exp(-age_days * ln(2) / half_life)`,  
`age_days` – возраст курса в днях.

**Подобранные экспериментально параметры (результат интерактивного подбора):**

| Параметр       | Значение | Обоснование                                                                 |
|----------------|----------|-----------------------------------------------------------------------------|
| `w_views`      | 0.05     | Минимальное влияние – просмотры легко накрутить и подвержены кликбейту      |
| `w_likes`      | 0.75     | Наибольший вес – лайки являются лучшим сигналом реальной полезности         |
| `w_novelty`    | 0.20     | Умеренный бонус за свежесть, чтобы новые курсы не вытесняли старые хиты     |
| `half_life`    | 180 дней | Период полураспада новизны: старые качественные курсы долго остаются в топе |
| `scale`        | 1000     | Множитель для приведения `novelty_score` к масштабу просмотров/лайков       |

## Входные данные

Модуль ожидает данные о курсах в формате списка словарей (можно загружать из БД через `load_courses_from_db`):

```python
[
    {
        "id": 123,
        "views": 1500,
        "likes": 320,
        "created_at": date(2025, 1, 15)
    },
    ...
]
```
# Модуль популярностного отбора Top‑N

## Выходные данные

Метод `get_top_n_courses(top_n=10, ...)` возвращает словарь:

```python
{
    "course_ids": [101, 205, ...],          # список ID top‑N курсов
    "total": 10,                            # количество возвращённых курсов
    "weights_used": {...},                  # использованные веса
    "half_life_days": 180,                  # период полураспада
    "scale": 1000,                          # множитель новизны
    "timestamp": "2025-04-01"               # дата расчёта
}
```
## Как запускать

### 1. Интерактивный подбор весов

Используйте ноутбук `weight_experiments_top_n.ipynb`. Он содержит функцию `interactive_weight_input()`, которая:

- запрашивает у пользователя веса и параметры;
- вычисляет веса курсов;
- выводит топ‑5 курсов и детализацию (просмотры, лайки, новизна).

**Запуск:**

```bash
jupyter notebook weight_experiments_top_n.ipynb
# или
python weight_experiments_top_n.py
```
# Настройка весов (пример ввода в консоли)

В консоли введите значения (для выхода – q):
Вес просмотров (0-1): 0.05
Вес лайков (0-1): 0.75
Вес новизны (0-1): 0.20
Период полураспада (дней, Enter=90): 180
Множитель новизны (scale, Enter=1000): 1000


## Программное использование в production

```python
from top_n import CourseRecommender

recommender = CourseRecommender(db_session=your_async_session)  # если данные в БД
# или передайте свой список курсов напрямую
recommender.courses_cache = your_list_of_courses

result = await recommender.get_top_n_courses(
    top_n=10,
    weights={'views': 0.05, 'likes': 0.75, 'novelty': 0.20},
    half_life_days=180,
    scale=1000
)
print(result["course_ids"])
```

## Результаты экспериментов
* На тестовом наборе из 12 курсов (разные по возрасту, просмотрам и лайкам) выбранная конфигурация показала:

* Устойчивость к кликбейту: курс с 10000 просмотров и 0 лайков оказался внизу списка, несмотря на высокие просмотры.

* Подъём новых качественных курсов: курс с возрастом 10 дней, 500 просмотрами и 100 лайками попал в топ‑3.

* Сохранение старых хитов: курс с возрастом 500 дней, 5000 просмотрами и 800 лайками остался лидером.

* Отодвигание непопулярных курсов: старый курс с низкими просмотрами и лайками оказался в самом низу.
