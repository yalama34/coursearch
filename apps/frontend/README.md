## Ответственность

- Отображение пользовательского интерфейса платформы
- Аутентификация и управление сессией пользователя
- Взаимодействие с backend API (core-api)
- Сбор поведенческих метрик (просмотры, лайки, время на странице)

## Запуск
А) Через Docker Compose:
1. Запуск в `docker-compose.yaml` / `docker-compose.dev.yaml`

Б) Через npm:
1. Установка зависимостей: `npm install`
2. Локальный запуск: `npm run dev` (Vite dev-server на порту 5173)
3. Сборка для продакшена: `npm run build` → артефакты в `dist/`
4. Запуск тестов: `npm run test` (Vitest + React Testing Library)

Конфигурация окружения задаётся в `.env`

## Архитектура

### Pages

`src/components/`

Страницы приложения, соответствующие маршрутам, и стили для каждой для отрисовки интерфейса. Каждая страница импортирует хуки для получения данных. Отвечают за композицию блоков, обработку состояний загрузки/ошибок и навигацию. Не содержат бизнес-логики - получают готовые данные из `hooks/`.

### Hooks

`src/hooks/`

Кастомные хуки инкапсулируют жизненный цикл асинхронных операций:

- `useAuth` - управление сессией, логин/регистрация/выход
- `useProfile` - загрузка профиля
- `useCourse` - загрузка  информации о курсе
- `useCourseTracking` - отправка событий `view`, `like`, `engagement`
- `useRecommendations` - загрузка рекомендаций для пользователя

Хуки возвращают унифицированный интерфейс `{ data, isLoading, error, refetch }`, что упрощает обработку состояний в компонентах.

### Services

`src/services/`

Низкоуровневые функции для работы с HTTP. Отвечают за:

- Получение и обработку данных из бэкенда
- Проверку авторизации пользователя и формирование Headers

Не содержат состояния, вызываются только через хуки.

### Types

`src/types/`

TypeScript-интерфейсы, описывающие контракты API и внутреннюю структуру данных. Все типы строго типизированы, что позволяет отлавливать ошибки на этапе компиляции. Примеры: `ProfileData`, `Course`, `AuthResponse`, `ActionPayload`.

### Routing

`src/App.tsx`

Декларативная конфигурация маршрутов через `react-router-dom`. Защищённые маршруты (`/`, `/profile`, `/course/:id`, `/setup-interests`) проверяют наличие валидного токена перед рендером. Публичные маршруты (`/welcome`, `/login`, `/register`) доступны без аутентификации. Активная вкладка в header синхронизируется с текущим `pathname`.
